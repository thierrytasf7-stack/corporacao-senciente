# Model-2-Vibrant-Tech: Voz Corporativa (Eutaxia Tone)

## 1. TECHNICAL SPECIFICATION (The Language)

### 1.1 Tone & Voice
- **Persona:** Inovadora, Assertiva, mas Acessível.
- **Style:** Frases curtas, foco em benefícios técnicos e progresso.
- **Tone:** Minimalista. Se uma palavra não comunica valor, remova-a.

### 1.2 Copy Rules
- Use o imperativo para botões: "Gerar Relatório", "Sincronizar Dados".
- Feedbacks de erro devem ser propositivos: "Conexão interrompida. Tentar reconectar?" em vez de apenas "Erro de conexão".

## 2. IMPLEMENTATION GUIDE (Style Guide)

| Contexto | Padrão |
| --- | --- |
| **Sucesso** | "Operação Concluída com Sucesso" |
| **Aguardando** | "Processando Fluxo de Dados..." |
| **Empty State** | "Inicie um novo projeto para ver os dados aqui." |

## 3. GUARDRAILS (A LEI)
- **REGRA 1:** NUNCA use gírias ou termos excessivamente informais. A seriedade técnica é inegociável.
- **REGRA 2:** TODO termo técnico complexo deve ter um tooltip explicativo opcional.
- **REGRA 3:** O texto deve ser sempre legível sobre o fundo cinza 900 (Contraste AAA).

## 4. COMPLIANCE CHECKLIST
- [ ] O tom de voz é consistente entre o Dashboard e o Login?
- [ ] Não existem mensagens de erro genéricas sem solução proposta?
