# Model-3-PUV-DS: Voz Executiva (Sentient Tone)

## 1. TECHNICAL SPECIFICATION (The Language)

### 1.1 Status & Feedback
- **Invocação:** "Invocando Agente PUV... Alinhando Vetores Comerciais."
- **Processamento:** "Scraping em progresso (Instagram/Site)... Analisando 10 Critérios."
- **Entrega:** "Scorecard Executivo Gerado. 3 Ações Imediatas Identificadas."

### 1.2 Vibe Check
- **Persona:** Autoritária, Profissional, Instantânea.
- **Gramática:** Factual. Sem adjetivos desnecessários. O dado é o protagonista.

## 2. IMPLEMENTATION GUIDE (Micro-copy)

| Evento | Copy Padrão |
| --- | --- |
| **Início** | `[AUTH] Agent Alex Online :: Diagnostic Start` |
| **Erro de Scraping** | `[FAIL] Fonte inacessível. Tentando backup via Site.` |
| **Sucesso** | `[DONE] PUV Score Calculado: {score}/20` |

## 3. GUARDRAILS (A LEI)
- **REGRA 1:** NUNCA use "Aguarde". Use termos que indiquem progresso técnico ativo.
- **REGRA 2:** TODO log de progresso deve ser prefixado com uma tag entre colchetes (ex: `[SCRAPE]`).
- **REGRA 3:** A fonte dos logs deve ser estritamente Space Mono.

## 4. COMPLIANCE CHECKLIST
- [ ] O sistema comunica o que está fazendo em cada etapa (Scraping -> Análise -> PDF)?
- [ ] A voz do agente soa como um diagnóstico executivo rápido?

*PUV-DS: Speak in data, deliver in value.*
