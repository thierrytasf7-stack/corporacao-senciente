## Contexto
Componentes UI sem testes podem quebrar silenciosamente durante refatoracoes. Snapshot tests servem como rede de seguranca automatica, detectando qualquer mudanca visual acidental antes de chegar em producao.

## Objetivo
Componentes UI em apps/dashboard/src/components/ui/ nao tem cobertura de testes. Criar snapshot tests para garantir que mudancas visuais sejam detectadas automaticamente.

## Instrucoes
1. Configurar Jest com suporte a snapshot testing no dashboard. 2. Criar arquivo de teste por componente em apps/dashboard/tests/components/. 3. Renderizar cada componente com props padroes e gerar snapshot. 4. Documentar processo de atualizacao de snapshots no README do dashboard.

## Exemplo de Referencia
```
import { render } from '@testing-library/react';
import { Button } from '@/components/ui/button';

describe('Button', () => {
  it('renders default state', () => {
    const { container } = render(<Button>Click me</Button>);
    expect(container).toMatchSnapshot();
  });

  it('renders disabled state', () => {
    const { container } = render(<Button disabled>Disabled</Button>);
    expect(container).toMatchSnapshot();
  });
});
```
