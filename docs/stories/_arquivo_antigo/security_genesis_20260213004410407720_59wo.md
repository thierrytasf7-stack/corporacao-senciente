## Contexto
Sem CSP, o dashboard e vulneravel a XSS - scripts maliciosos podem ser injetados e executados no contexto do usuario. Como o dashboard tem acesso a dados dos agentes e do pipeline, protege-lo contra XSS e critico para a seguranca do sistema.

## Objetivo
O dashboard Next.js nao define Content Security Policy headers, permitindo execucao de scripts de terceiros e XSS. Configurar CSP headers restritivos no next.config.js.

## Responsavel
**✅ Quinn** (Test Architect & QA) e o agente AIOS responsavel por esta task.
- **Foco:** Quality gates, test architecture, code review, validacao de criteria
- **Invocacao:** `/AIOS:agents:qa`
- **Executor runtime:** `@agente-zero` (worker autonomo que processa a story)

- `apps/dashboard/src/app/`
- `apps/dashboard/`

## Output Esperado
- next.config.js com headers() retornando Content-Security-Policy restritivo
- CSP permitindo 'self' para scripts, styles, imagens e fontes
- Middleware ou layout gerando nonce para scripts inline necessarios
- Console do browser sem erros de CSP violation ao navegar no dashboard

## Acceptance Criteria
- [?] CSP headers configurados no next.config.js
- [?] Scripts inline bloqueados (usando nonce quando necessario)
- [?] Styles e imagens restritos a origens conhecidas
- [?] Dashboard funcional sem violacoes de CSP

## Constraints
- CSP nao deve bloquear hot reload em modo desenvolvimento (next dev)
- Usar report-only primeiro para validar, depois trocar para enforce
- Nao usar 'unsafe-inline' ou 'unsafe-eval' no modo producao
- NAO modificar arquivos fora do escopo listado em Arquivos Alvo
- NAO introduzir dependencias externas sem justificativa
- NAO quebrar funcionalidades existentes

## Instrucoes
1. Definir CSP policy restritiva em next.config.js headers. 2. Permitir self para scripts, styles e imagens. 3. Adicionar nonce para scripts inline necessarios. 4. Testar dashboard completo verificando console por violacoes CSP.

## Exemplo de Referencia
```
// next.config.js
module.exports = {
  async headers() {
    return [{
      source: '/(.*)',
      headers: [{
        key: 'Content-Security-Policy',
        value: [
          "default-src 'self'",
          "script-src 'self' 'nonce-${nonce}'",
          "style-src 'self' 'unsafe-inline'",
          "img-src 'self' data: blob:",
          "connect-src 'self' ws://localhost:*",
          "font-src 'self'",
        ].join('; ')
      }]
    }];
  },
};
```
