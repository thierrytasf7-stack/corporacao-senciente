## Contexto
Sem health check, PM2 nao consegue reiniciar o monitor-server automaticamente quando ele trava. Isso causa perda silenciosa de observabilidade, violando o principio de que o sistema deve operar 24/7 sem intervencao manual.

## Objetivo
O monitor-server nao possui endpoint de health check, dificultando monitoramento por PM2 e scripts externos. Criar endpoint /health que retorne status dos subsistemas: websocket, filesystem watcher e conexao com dashboard.

- **Foco:** CI/CD, git operations, releases, infraestrutura e deploy

- **Executor runtime:** `@aider` (worker autonomo que processa a story)

## Output Esperado

- ecosystem.config.js atualizado com health_check_url apontando para /health
- Resposta JSON incluindo: uptime, ws_connections, fs_watcher_status, memory_usage
- Tempo de resposta do endpoint abaixo de 100ms verificado com console.time

- NAO quebrar funcionalidades existentes

## Instrucoes
1. Criar rota /health no monitor-server com Bun.serve. 2. Verificar status de cada subsistema: ws connections, fs watcher, uptime. 3. Retornar JSON estruturado com status individual e geral. 4. Atualizar ecosystem.config.js para usar health check URL. 5. Testar manualmente com curl e validar resposta JSON.

## Exemplo de Referencia
```
// Bun.serve health check handler
if (url.pathname === '/health') {
  const health = {
    status: 'healthy',
    uptime: process.uptime(),
    wsConnections: wsClients.size,
    fsWatcher: watcher.isActive(),
    memoryMB: Math.round(process.memoryUsage().heapUsed / 1024 / 1024),
    version: pkg.version,
    timestamp: new Date().toISOString(),
  };
  const httpStatus = health.fsWatcher ? 200 : 503;
  return new Response(JSON.stringify(health), { status: httpStatus });
}
```
