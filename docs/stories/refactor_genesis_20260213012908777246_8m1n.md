## Contexto
Stores monoliticas se tornam dificeis de manter e testar conforme o dashboard cresce. O padrao slices do Zustand permite que cada dominio evolua independentemente, facilita testes unitarios e reduz conflitos de merge.

## Objetivo
As stores Zustand do dashboard estao monoliticas com toda a logica em um arquivo. Refatorar para o padrao slices, separando concerns em fatias independentes que sao compostas na store principal.

## Constraints
- Seguir pattern de slices do Zustand v4+ (StateCreator com generics)

- Nao alterar a API publica das stores - apenas reorganizacao interna
- NAO modificar arquivos fora do escopo listado em Arquivos Alvo
- NAO introduzir dependencias externas sem justificativa
- NAO quebrar funcionalidades existentes

## Instrucoes
1. Mapear a store atual e identificar dominios: agents, workflows, settings, ui. 2. Criar uma slice por dominio em apps/dashboard/src/stores/slices/. 3. Compor as slices na store principal usando spread pattern do Zustand. 4. Atualizar imports nos componentes que consomem a store. 5. Rodar testes e validar que nada quebrou.

## Exemplo de Referencia
```
import { StateCreator } from 'zustand';

export interface AgentSlice {
  agents: Record<string, Agent>;
  setAgents: (agents: Agent[]) => void;
}

export const createAgentSlice: StateCreator<AgentSlice> = (set) => ({
  agents: {},
  setAgents: (agents) => set({
    agents: Object.fromEntries(agents.map(a => [a.id, a]))
  }),
});

// Main store composition
const useStore = create<AgentSlice & UiSlice>()((...a) => ({
  ...createAgentSlice(...a),
  ...createUiSlice(...a),
}));
```
