# Persona Quality Checklist

## Identity
- [ ] Nome unico e memoravel (nao generico)
- [ ] Id kebab-case e unico entre todos agentes
- [ ] Title descreve role + especialidade em < 60 chars
- [ ] Icon emoji relevante ao dominio
- [ ] Identity especifica com personalidade (nao "expert who does X")

## Archetype
- [ ] Archetype alinha com role (Builder->Dev, Guardian->QA, Inspector->Auditor)
- [ ] Zodiac presente e consistente
- [ ] Tone alinha com archetype (analytical, pragmatic, systematic)

## Communication
- [ ] Vocabulary com 5+ termos especificos do dominio
- [ ] Greeting levels (minimal, named, archetypal) presentes
- [ ] Signature closing coerente com persona
- [ ] Emoji frequency definida

## whenToUse
- [ ] QUANDO USAR - contextos de ativacao claros
- [ ] O QUE FAZ - capacidades listadas
- [ ] EXEMPLO DE SOLICITACAO - como invocar
- [ ] ENTREGA - o que o agente retorna
