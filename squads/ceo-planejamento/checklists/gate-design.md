# Quality Gate: Design Phase

## Mandatory (all must pass)
- [ ] User flows mapped for key interactions
- [ ] Wireframes created (appropriate fidelity)
- [ ] Frontend specification document generated
- [ ] Responsive strategy defined

## UX Excellence (4+ must pass)
- [ ] User personas referenced in design decisions
- [ ] Error states designed for all operations
- [ ] Loading states and skeleton screens planned
- [ ] Empty states designed
- [ ] Success/confirmation states designed
- [ ] Navigation structure intuitive
- [ ] Information hierarchy clear

## UI Polish (4+ must pass)
- [ ] Design system tokens defined (or existing system referenced)
- [ ] Typography scale consistent
- [ ] Color palette with semantic colors
- [ ] Spacing system (4/8px grid)
- [ ] Component inventory (atomic design)
- [ ] Dark mode consideration
- [ ] Animation/transition guidelines

## Accessibility (all must pass)
- [ ] WCAG AA color contrast ratios
- [ ] Keyboard navigation planned
- [ ] Screen reader flow defined
- [ ] Focus management strategy
- [ ] Touch targets >= 44px

## Gate Decision
- **PASS**: All mandatory + accessibility checks pass, 4+ in UX and UI
