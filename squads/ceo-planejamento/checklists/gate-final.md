# Quality Gate: Final Validation

## Cross-Phase Consistency (all must pass)
- [ ] PRD objectives fully covered by stories
- [ ] Architecture supports all planned features
- [ ] Design spec aligns with architecture patterns
- [ ] Stories reference correct architecture components
- [ ] Complexity estimates consistent with architecture
- [ ] Dependencies between stories properly ordered
- [ ] No contradictions between phase outputs

## Quality Scorecard (minimum: average >= 7)
- [ ] Performance: >= 7/10
- [ ] Scalability: >= 7/10
- [ ] Security: >= 8/10 (higher bar)
- [ ] UX Excellence: >= 7/10
- [ ] UI Polish: >= 7/10
- [ ] Accessibility: >= 7/10
- [ ] Maintainability: >= 6/10
- [ ] Testability: >= 6/10
- [ ] Cost Efficiency: >= 5/10
- [ ] Time to Market: >= 6/10

## Masterplan Completeness (all must pass)
- [ ] Executive summary written
- [ ] All artifacts consolidated
- [ ] Risk assessment documented
- [ ] Implementation roadmap (phased)
- [ ] Story sequence for @dev clear
- [ ] Handoff instructions complete

## Constitution Compliance (all must pass)
- [ ] CLI First respected throughout
- [ ] Port policy followed
- [ ] No invention beyond requirements
- [ ] Story-driven development followed
- [ ] Quality gates embedded in stories

## Gate Decision
- **APPROVED**: All checks pass → Masterplan ready for execution
- **REVISION**: Quality score < 7 in any critical dimension → iterate
- **BLOCKED**: Cross-phase inconsistency → must resolve before delivery
