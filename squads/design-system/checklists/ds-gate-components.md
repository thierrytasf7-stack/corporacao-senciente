# Quality Gate: Component Specifications

## Per Component
- [ ] TypeScript props interface defined
- [ ] All variants documented with use cases
- [ ] All sizes documented with dimensions
- [ ] All states defined (default, hover, active, focus, disabled, loading)
- [ ] Tokens mapped (no hardcoded values)
- [ ] Keyboard navigation specified
- [ ] ARIA attributes defined
- [ ] Usage examples with code
- [ ] Do/Don't patterns documented
- [ ] Composition examples (if composite)

## Overall
- [ ] Tier 1 (Primitives/Atoms) complete
- [ ] Naming follows PascalCase convention
- [ ] Props API consistent across similar components
- [ ] Size scale consistent (sm/md/lg across all sized components)
- [ ] Variant naming consistent (primary/secondary/ghost)
