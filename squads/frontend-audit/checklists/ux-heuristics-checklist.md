# UX Heuristics Checklist (Nielsen's 10)

## H1 - Visibilidade do Status do Sistema
- [ ] Loading indicators em operacoes assincronas
- [ ] Progress bars em processos multi-step
- [ ] Confirmacao visual apos acoes do usuario
- [ ] Breadcrumbs ou indicacao de localizacao
- [ ] Estado ativo no menu de navegacao
- [ ] Contadores/badges atualizados em tempo real

## H2 - Correspondencia Sistema-Mundo Real
- [ ] Linguagem familiar (sem jargao tecnico desnecessario)
- [ ] Icons reconheciveis e padrao da industria
- [ ] Ordem logica de informacoes
- [ ] Metaforas visuais consistentes
- [ ] Datas e numeros em formato local

## H3 - Controle e Liberdade do Usuario
- [ ] Botao de cancelar em dialogos/modais
- [ ] Undo disponivel apos acoes destrutivas
- [ ] Saida clara de qualquer fluxo
- [ ] Back button do browser funciona corretamente
- [ ] Filtros e buscas podem ser limpos facilmente

## H4 - Consistencia e Padroes
- [ ] Botoes tem estilo consistente por tipo (primary, secondary, danger)
- [ ] Mesma acao usa mesmo padrao em todas as paginas
- [ ] Terminologia consistente (nao alterna entre termos)
- [ ] Layout grid consistente entre paginas
- [ ] Cores tem significado consistente (success=green, error=red)

## H5 - Prevencao de Erros
- [ ] Validacao inline em formularios (antes de submit)
- [ ] Confirmacao antes de acoes destrutivas (delete)
- [ ] Campos obrigatorios claramente marcados
- [ ] Formato esperado indicado (placeholder, helper text)
- [ ] Botoes desabilitados quando acao nao e possivel

## H6 - Reconhecimento vs Memoria
- [ ] Opcoes visiveis (nao dependem de memoria do usuario)
- [ ] Tooltips em elements complexos
- [ ] Contexto mantido durante navegacao
- [ ] Recentes/favoritos acessiveis
- [ ] Labels claros em todos os campos

## H7 - Flexibilidade e Eficiencia
- [ ] Atalhos de teclado para acoes frequentes
- [ ] Busca/filtro rapido disponivel
- [ ] Acoes em batch quando aplicavel
- [ ] Customizacao de interface (dark mode, layout)
- [ ] Acoes frequentes sao acessiveis em < 3 cliques

## H8 - Design Estetico e Minimalista
- [ ] Apenas informacao relevante na tela
- [ ] Hierarquia visual clara
- [ ] Espacamento adequado (nao claustrofobico)
- [ ] Uso de whitespace efetivo
- [ ] Sem elementos decorativos que distraem

## H9 - Recuperacao de Erros
- [ ] Mensagens de erro em linguagem simples
- [ ] Erro indica O QUE deu errado
- [ ] Erro indica COMO resolver
- [ ] Botao de retry quando aplicavel
- [ ] Estado anterior preservado apos erro

## H10 - Ajuda e Documentacao
- [ ] Tooltips em elementos nao-obvios
- [ ] Help acessivel de qualquer pagina
- [ ] Onboarding para novos usuarios
- [ ] Exemplos em campos complexos
- [ ] FAQ ou knowledge base linkado
