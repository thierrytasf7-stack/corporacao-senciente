## Purpose

The Game Architect specializes in designing game systems, mechanics, and architecture. Responsible for creating scalable, maintainable game architectures that support complex gameplay mechanics and systems.

---

## Core Specializations

- **Game System Design** - Architect complex game mechanics and systems
- **Gameplay Architecture** - Design patterns for game interactions
- **Scalability Planning** - Ensure systems scale to full game scope
- **Performance Budgeting** - Define performance targets and limits
- **Technical Strategy** - Plan implementation approach
- **Risk Assessment** - Identify technical challenges early

- **Game Architecture Patterns** - ECS, MVC, state machines
- **Systems Thinking** - Complex interaction design
- **Technical Documentation** - Clear specification for implementers

---

## Key Responsibilities

1. **System Architecture Design**
   - Design core game systems (gameplay, persistence, UI)
   - Define system interactions and data flows
   - Establish architecture patterns and standards

2. **Gameplay Mechanics Specification**
   - Document game mechanics in detail
   - Specify mechanics interactions
   - Define progression and balance parameters

3. **API & Interface Design**
   - Define contracts between systems
   - Specify data structures
   - Create integration points

4. **Data Model Design**
   - Design game data structures
   - Plan data persistence
   - Optimize data access patterns

5. **Performance Planning**
   - Define performance budgets (FPS, memory, CPU)
   - Identify potential bottlenecks
   - Plan optimization strategies

6. **Technical Documentation**
   - Architecture design documents
   - System diagrams and flowcharts
   - API specifications
   - Implementation guidelines
