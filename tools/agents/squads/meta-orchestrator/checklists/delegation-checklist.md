# Delegation Checklist

## Pre-Delegation Validation

### Task Understanding
- [ ] Domain clearly identified
- [ ] Requirements extracted
- [ ] Complexity assessed (simple/medium/complex)
- [ ] Urgency determined
- [ ] Expected outputs defined

### Squad Analysis
- [ ] All relevant squads scanned
- [ ] Capability scores calculated
- [ ] Best match identified
- [ ] Gaps documented
- [ ] Alternative options listed

### Routing Decision
- [ ] Decision threshold applied (â‰¥70% = delegate)
- [ ] Justification documented
- [ ] User informed of routing choice
- [ ] Fallback plan established

## Delegation Execution

### Task Handoff
- [ ] Task formatted for target squad
- [ ] Context provided to receiving agent
- [ ] Dependencies resolved
- [ ] Timeline communicated

### Monitoring Setup
- [ ] Sentinel notified
- [ ] Tracking ID assigned
- [ ] Quality gates defined
- [ ] Alert thresholds set

## Post-Delegation

### Execution Tracking
- [ ] Progress monitored
- [ ] Issues detected early
- [ ] Interventions triggered if needed
- [ ] Status reported to user

### Completion & Learning
- [ ] Output validated
- [ ] Quality checked
- [ ] Outcome recorded by Cortex
- [ ] Patterns extracted
- [ ] Routing intelligence updated

## Quality Gates

| Gate | Criteria | Status |
|------|----------|--------|
| QG-001 | Task understood | [ ] |

## Red Flags

Stop delegation if:
- [ ] Task ambiguous after clarification attempts
- [ ] No squad scores above 30%
- [ ] Security concerns identified
- [ ] User explicitly requests hold
