# Sector Templates

## Legal Sector Templates

### Legal Research Template
```yaml
template:
  name: legal-research-template
  type: markdown
  sections:
    - case-summary
    - legal-analysis
    - precedent-research
    - compliance-check
    - risk-assessment
  placeholders:
    {case-name}: Name of the legal case
    {jurisdiction}: Legal jurisdiction
    {relevant-laws}: Applicable laws
    {precedents}: Relevant case precedents
    {compliance-requirements}: Regulatory requirements
```

## Template Usage Guidelines

1. **Sector-Specific Customization**: Each template should be customized for the specific sector requirements
2. **Placeholder Examples**: Include realistic examples for each placeholder
3. **Validation Criteria**: Add sector-specific validation criteria
4. **Integration Points**: Document how templates integrate with sector workflows
5. **Best Practices**: Include sector-specific best practices for template usage

## Pattern Library Integration

All sector templates follow SC-A pattern naming:
- SC-A-013: Sector Template Standardization
- SC-A-014: Sector-Specific Elicitation
- SC-A-015: Sector Validation Criteria
- SC-A-016: Sector Integration Patterns
